Implement “Server-only Solana RPC” on mainnet (Option A). Remove all direct client-side RPC usage.

Goal:
- All Solana reads/writes must go through server/services/rpcService.ts (Helius + fallback).
- Frontend must NOT create a Solana Connection to public RPCs on mainnet.
- This keeps Helius centralized and prevents API key exposure and inconsistent RPC behavior.

Required changes:

1) Remove/disable client RPC on mainnet
- Identify all imports/usages of:
   client/src/lib/solana.ts
   new Connection(...)
   connection.get* calls in client
- Refactor client components/hooks to call server endpoints instead.
- client/src/lib/solana.ts should not be used for mainnet reads at all.
- It may remain for devnet/local testing only, gated by env flag.

2) Add/extend server API endpoints for any client reads currently done via client Connection
Examples (implement only what the client currently needs):
- GET /api/wallet/balances?owner=<pubkey>
- GET /api/wallet/sol-balance?owner=<pubkey>
- GET /api/wallet/token-accounts?owner=<pubkey>
- GET /api/token/supply?mint=<mint>
- GET /api/token/largest-accounts?mint=<mint>  (if needed for UI)
- GET /api/tx/status?signature=<sig>
All endpoints must use rpcService (Helius).

3) Update client to use these endpoints
- Replace direct `connection.getBalance`, `getParsedTokenAccountsByOwner`, etc. with fetches to /api/...
- Use React Query caching / staleTime where appropriate (avoid spamming server).

4) Security & correctness constraints
- Do NOT expose Helius RPC URL or API key to the client.
- Maintain existing auth/session handling.
- Validate inputs with Zod (pubkeys, mints, signatures).
- Rate limit these endpoints (reuse existing limiter).
- Return errors in the existing consistent error format.

5) Developer visibility
- Add dev-only response headers on these endpoints:
   X-RPC-HOST: <hostname>
   X-RPC-TIER: <tier if applicable>
- Add a log line when rpcService selects an endpoint.

Deliverables:
- No client-side Solana Connection used on mainnet.
- All wallet/token reads originate from /api/ endpoints using Helius.
- App behavior remains the same but performance and consistency improve.
